Documentación del ToDo* -  Ver tareas pendientes/terminadas. Se debe realizar una solicitud por GET al endpoint https://app.co/v1/tasks con un token JWT valido (obtenido en la autenticación). Se debe parsear el JSON teniendo en cuenta los parámetros si son tareas pendientes o terminadas y posteriormente su renderización para la presentación en el front. La función deberá recibir un array con las tareas a mostrar getTareas(tareas), existirá otra función que se encargue de renderizar las tareas en el contenedor  de tareas pendientes o terminadas.* -  Marcar una tarea como terminada. Se debe realizar una solicitud mediante el método PUT para actualizar la tarea, para realizar esta acción se debe estar autenticado, es decir contar con un token valido.La función deberá recibir el ID de la tarea y el ESTADO. actualizarTarea(idTarea, estado)* -  Crear nuevas tareas. Se debe realizar una solicitud mediante el método POST al endpoint https://app.co/v1/tasks. En el body se debe enviar los datos de la tarea que serán usados en la función crearTarea(tarea)* -  Ingresar a nuestra cuenta (login). Se debe realizar una solicitud POST al enpoint https://app.co/v1/login pasando los parámetros de usuario y clave en el body, si los valores son ingresador debe retornar un token JWT que será requerido en los diferentes endpoints de nuestra aplicación.* -  Ver la fecha de creación de una tarea. En el template creado por la función de renderizar se debe parsear la fecha y darle formato DD/MM/AAAA
